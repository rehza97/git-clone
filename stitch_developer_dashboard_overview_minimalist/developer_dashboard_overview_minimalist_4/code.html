Archiving for Decision 1275

A complete guide on how to properly format, upload, and archive your academic project source code to comply with Ministerial Decision 1275.

Updated 2 days ago ~ 5 min read

Prerequisites

Before you begin the archiving process on the ASCAP platform, ensure your project meets the following criteria:

  • You have a valid student or researcher account on ASCAP.
  • Your project source code is hosted on a Git repository (local or remote).
  • You have generated a README.md file explaining the project structure.
warning

Important Compliance Notice

Ensure no sensitive data (API keys, passwords, database credentials) is included in your source code commits. The platform scans for secrets automatically and will reject insecure submissions.

Step 1: Preparing Your Repository

To ensure long-term preservation, we recommend structuring your repository cleanly. Use a .gitignore file to exclude build artifacts and temporary files.

Standard .gitignore configuration

# Node.js
node_modules/
dist/
.env
# Python
__pycache__/
*.pyc
venv/
# IDEs
.vscode/
.idea/

Step 2: Pushing to ASCAP Remote

Once your code is ready, you need to add the ASCAP remote repository URL to your local git configuration. You can find your specific repository URL in your dashboard. 

# Initialize git if you haven't already
git init
# Add all files
git add .
git commit -m "Initial submission for Decision 1275"
# Add ASCAP remote (replace with your unique URL)
git remote add ascap https://git.ascap.dz/username/project-slug.git
# Push to the archiving platform
git push -u ascap main

Step 3: Generating the Certificate

After a successful push, the system will automatically run static analysis. Once passed, navigate to the Certificates tab in your dashboard to download your official deposit proof.
